# ADR-002: Shared UI Component with Browser-Only Leaflet Loading

## Status

Accepted

## Date

2026-06-14

## Context

The Proxi frontend is an Angular 21 standalone application with server-side rendering enabled. Leaflet depends on browser DOM APIs, so direct static imports and map initialization during server rendering can fail. The map also needs to be reusable beyond the current search page without owning route navigation.

## Decision

Implement `ProxiMapComponent` under `apps/frontend/src/app/shared/components/ui/proxi-map` as a standalone UI component. The component will dynamically import Leaflet only in the browser after the view is initialized, guarded by Angular platform checks.

The component will receive normalized inputs for user location and providers, manage Leaflet map lifecycle internally, and emit provider profile intent through an output event.

## Alternatives Considered

### Alternative 1: Customer Search Local Component

- **Description**: Place the map under `modules/customer/search/components/proxi-map`.
- **Pros**: Keeps the implementation close to its first consumer.
- **Cons**: Reduces reuse and couples map behavior to the search page.
- **Why rejected**: The PRD expects a reusable map capability that can support future discovery contexts.

### Alternative 2: Static Leaflet Imports

- **Description**: Import Leaflet at the top of the component file and initialize normally.
- **Pros**: Simpler code and easier typing.
- **Cons**: Can break SSR because Leaflet expects browser globals.
- **Why rejected**: The application already uses SSR, so browser-only loading is required.

## Consequences

### Positive

- Prevents Leaflet from executing during server rendering.
- Keeps the map reusable and decoupled from route navigation.
- Matches existing standalone component patterns.

### Negative

- Dynamic imports add a small amount of lifecycle complexity.
- Unit tests must mock asynchronous Leaflet loading and browser guards.

### Risks

- The map may not initialize if the container is not visible or sized. Mitigation: invalidate map size after initialization and when inputs change.

## Implementation Notes

The component should use separate TypeScript, HTML, and SCSS files, follow the project's standalone component pattern, and keep Leaflet-specific DOM work inside guarded lifecycle methods.
