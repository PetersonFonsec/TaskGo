# ADR-005: Comprehensive Backoffice Verification Gate

## Status

Accepted

## Date

2026-07-02

## Context

The Backoffice controls privileged operations and must demonstrate role isolation, complete auditability, correct provider transitions, accessible frontend behavior, and responsive queue and audit queries.

## Decision

Require unit tests, PostgreSQL-backed integration tests, API E2E tests for all role and transition combinations, Angular component tests, Cypress critical journeys, and repeatable load checks. Paginated provider and audit list endpoints must remain below 500 ms at p95 under the agreed MVP load profile.

## Alternatives Considered

### Alternative 1: Balanced Functional Suite

- **Description**: Run unit, integration, and critical E2E tests without formal load checks.
- **Pros**: Faster implementation and CI execution.
- **Cons**: Leaves the dashboard and audit-query performance requirement unverified.
- **Why rejected**: The user selected an explicit performance gate.

### Alternative 2: Lean Unit and Smoke E2E Suite

- **Description**: Focus on unit tests and a small number of end-to-end journeys.
- **Pros**: Lowest maintenance cost.
- **Cons**: Insufficient evidence for permission-matrix and transactional-audit guarantees.
- **Why rejected**: Privileged workflows require broader negative-path coverage.

## Consequences

### Positive

- Authorization regressions are tested at the API boundary.
- Transaction and migration behavior is exercised against PostgreSQL.
- User journeys, accessibility states, and performance targets have explicit evidence.

### Negative

- CI takes longer and requires a database plus a load-test environment.
- Test fixtures and role matrices require ongoing maintenance.

### Risks

- Unstable E2E data can create flaky tests; use deterministic factories and isolated database resets.
- An undefined load profile can make p95 results meaningless; document concurrency, dataset size, warm-up, and duration before accepting results.

## Implementation Notes

Reuse Jest, the existing E2E harness, Angular tests, and Cypress. Select a load runner during task decomposition only if no existing workspace tool satisfies the documented profile.
