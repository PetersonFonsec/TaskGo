# ADR-001: Adopt Unified Adaptive Navigation

## Status

Accepted

## Date

2026-07-23

## Context

Authenticated users can currently access the full account navigation from a desktop sidebar, but that sidebar is hidden on mobile. This leaves mobile users without a reliable way to move between account areas. The navigation contains more destinations than a compact persistent mobile bar can comfortably expose, while users still need the same destinations and actions across screen sizes.

## Decision

Adopt a unified adaptive navigation experience. All authenticated screens will offer the same destination groups, labels, ordering, availability rules, active-state feedback, and account actions. The presentation will adapt to the available screen size: navigation remains directly visible when space permits and is available through a clearly identified control on compact screens.

## Alternatives Considered

### Alternative 1: Universal Expandable Menu

- **Description**: Keep navigation collapsed behind a control on both mobile and desktop.
- **Pros**: Provides nearly identical interaction across devices and leaves more room for page content.
- **Cons**: Makes destinations less discoverable on desktop and adds an interaction before every navigation choice.
- **Why rejected**: It weakens desktop discoverability without adding enough user value.

### Alternative 2: Two-Layer Mobile Navigation

- **Description**: Keep a small set of frequent destinations visible on mobile and place the remaining destinations under a "More" option.
- **Pros**: Speeds up access to recurring destinations and preserves access to the complete menu.
- **Cons**: Requires an unvalidated priority hierarchy and creates a different information structure on mobile.
- **Why rejected**: The current product evidence does not establish which destinations deserve priority, and the approach conflicts with the desired cross-device consistency.

## Consequences

### Positive

- Authenticated users can reach every available destination from any screen size.
- Navigation labels, grouping, order, and availability remain predictable across devices.
- Desktop navigation retains strong discoverability.
- Mobile navigation uses screen space without permanently crowding page content.

### Negative

- Mobile users perform one additional interaction to reveal navigation.
- The two presentations require consistent product rules to avoid behavioral drift.

### Risks

- Users may overlook an icon-only menu control; mitigate this with a familiar symbol, an accessible label, and consistent placement.
- A long menu may feel dense on small screens; mitigate this by preserving clear grouping and readable touch targets.
- Users may lose context after opening the menu; mitigate this with visible current-destination feedback.

## Implementation Notes

The technical specification should define how the shared navigation rules are represented and how each screen-size presentation consumes them. The PRD governs user-visible consistency and outcomes, not the implementation mechanism.
