# ADR-001: Favorites — MVP (Save + dedicated list + search filter)

## Status

Accepted

## Date

2026-05-30

## Context

We need a scoped, user-facing decision that defines how the "favorites" capability will appear for clients. The product goal is to let clients mark professionals as favorites and quickly view only those professionals when searching or from a dedicated list. The decision must prioritize a minimal viable experience that delivers clear user value with low delivery risk.

## Decision

Implement an MVP favorites feature that provides:

- A persistent "favorite/unfavorite" control exposed on professional cards and profiles.
- A dedicated `Favorites` section accessible from the client's menu where they can view all favorited professionals.
- A filter toggle on search/results pages labeled "Show only favorites" that restricts results to favorited professionals for the signed-in client.
- Favorites persisted per client account and remembered across sessions.

The MVP will not include multi-list shortlists, sharing, or personalization-driven ranking. Those are deferred alternatives.

## Alternatives Considered

### Alternative 1: Enhanced shortlists and sharing

- Description: Allow clients to create multiple named lists (shortlists), add labels, and share lists with team members.
- Pros: Better for enterprise workflows and collaboration; higher perceived value for teams.
- Cons: Higher complexity, longer delivery time, additional privacy/permission work.
- Why rejected: Out of scope for an MVP; defers core value delivery and slows launch.

### Alternative 2: Integrated personalization and ranking

- Description: Use favorites to influence search ranking and personalized recommendations.
- Pros: Increased discovery and long-term engagement; stronger personalization.
- Cons: Requires tracking, modeling, and privacy decisions; risk of biasing search too early.
- Why rejected: Requires cross-team work and product metrics before safe rollout; deferred to a later phase.

## Consequences

### Positive

- Fast delivery of a useful feature that increases client engagement with the marketplace.
- Clear UX pattern (save + list + filter) that users understand from other marketplaces.

### Negative

- Limits collaboration and advanced workflows until future phases.

### Risks

- Users may expect cross-device sync or sharing—mitigate by clearly stating scope in product copy and UI.
- Potential for stale favorite lists if providers remove profiles; show graceful empty/error states.

## Implementation Notes

- Persist favorites per client identity; expose endpoints to list, add, and remove favorites.
- UI should clearly show favorite state on cards and profile headers; the search filter should combine with existing filters.
