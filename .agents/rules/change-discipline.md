# Change Discipline

- Read nearby code before editing.
- Match the existing style, naming, structure, and test patterns.
- Keep changes small and directly tied to the task.
- Do not perform drive-by refactors.
- Do not reformat unrelated files.
- Do not update dependencies unless the task requires it.
- Do not introduce abstractions until they remove real duplication or match an established local pattern.
- Preserve existing behavior unless the task asks to change it.
- If a bug fix changes behavior, update or add tests that demonstrate the intended behavior.
- Before finishing, summarize changed files and verification performed.
